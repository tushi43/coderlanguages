Software Development

Building a software is not just coding only, it is way more than that. Take a look at below image you will know what happens if product development is done without a structured manner.




Just make sure you do not do mistake like above stated.

  1. Talk with client as much as possible to gather requirements.
  2. Have a good analysis of project/product you are building.
  3. Build a better prototype
  4. Code using comments, proper naming conventions and use Object Oriented Design (OO Design) which makes maintenance easy.
  5. Do proper testing


I'm saying this because I have suffered in my college project. This above all steps of Information gathering, analysis, design, coding, testing, maintenance are very important. When I design a paper prototype for next project it was really easy to code for that project. I had not done any information gathering before for which the my project suffers. Now, I don't.
